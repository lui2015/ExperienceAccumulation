## 一、项目背景与目标

### 1.1 背景
个人在「互联网项目管理」「金融投资」「思维认知」三个领域持续学习与实践，过程中沉淀了大量经验性 HTML 文档（包括笔记、复盘、方法论等）。这些资料目前散落在本地或不同工具中，难以统一查阅、维护和回顾。

### 1.2 目标
搭建一个**轻量级的个人经验沉淀网站**，作为个人知识库的"第二大脑"，实现：
- 按主题领域分类沉淀（三大 Tab）
- 支持自行管理（增、删、改、排序）
- 支持上传现成的 HTML 文档并云端存储
- 即开即用，长期可维护

### 1.3 非目标（本期不做）
- 不做多人协作 / 团队权限（仅站主 + 受邀只读访客）
- 不做全文检索（一期靠 Tab + 列表浏览）
- 不做评论、点赞等社交功能
- 不做富文本在线编辑器（一期只支持上传 HTML）

## 二、用户与使用场景

### 2.1 目标用户与角色

| 角色 | 说明 | 权限 |
| --- | --- | --- |
| **站主（Owner）** | 网站拥有者，唯一 | 全部权限：登录、增删改查、拖拽排序、分类管理、上传文件 |
| **访客（Visitor）** | 受邀只读用户（朋友、同事等） | 只读：查看 Tab、查看卡片列表、打开 HTML 文档 |
| **游客（Guest）** | 未登录或未受邀用户 | 视配置而定：默认拒绝访问；或开启"公开只读"模式时同访客 |

> 一期实现：站主单账号 + 访客通过**邀请码 / 受邀邮箱白名单**登录。

### 2.2 核心使用场景

| 场景编号 | 角色 | 场景描述 |
| --- | --- | --- |
| S1 | 站主 | 学习了一个新方法论，整理成 HTML 后上传到对应 Tab 沉淀 |
| S2 | 站主 / 访客 | 想回顾「金融投资」中关于"仓位管理"的笔记，进入 Tab 找到对应卡片打开阅读 |
| S3 | 站主 | 重新整理知识结构，把高频复看的内容拖到列表前面 |
| S4 | 站主 | 发现旧文档信息过时，编辑标题/描述/重新上传新版本 HTML |
| S5 | 站主 | 不再需要某篇内容，直接删除 |
| S6 | 站主 | 学习领域扩展，新增第 4 个 Tab「读书笔记」，调整 Tab 顺序 |
| S7 | 站主 | 邀请朋友以只读账号查看自己的某些经验沉淀 |

## 三、功能需求

### 3.1 信息架构

```
经验沉淀网站
├── 登录页（账号密码 / 邀请码）
└── 主应用（登录后）
    ├── 顶部导航
    │   ├── Tab 区（动态渲染，可扩展）
    │   │   ├── Tab 1：互联网项目管理
    │   │   ├── Tab 2：金融投资
    │   │   ├── Tab 3：思维认知
    │   │   └── ...（站主可新增）
    │   ├── 「分类管理」入口（仅 Owner 可见）
    │   └── 用户菜单（头像 / 退出登录 / 访客管理）
    └── 每个 Tab 下
        ├── 经验卡片列表（支持拖拽排序）
        ├── 「+ 新增经验」按钮（仅 Owner 可见）
        └── 卡片操作：查看 / 编辑 / 删除（仅 Owner 可见）
```

### 3.2 功能清单

#### F0. 登录与权限（新增）
- **F0.1** 提供登录页，支持账号密码登录
- **F0.2** **站主账号**：通过环境变量初始化（用户名 + 密码哈希），唯一管理员
- **F0.3** **访客账号**：站主可在「访客管理」页面创建/启用/禁用访客账号
  - 字段：用户名、备注（如朋友姓名）、状态（启用/禁用）、初始密码（创建时一次性展示并复制）
  - 访客密码由系统生成 12 位随机串，站主复制后线下交付
- **F0.4** 登录态使用 **JWT（HttpOnly Cookie + SameSite=Strict）**，有效期 7 天，支持续签
- **F0.5** 登录失败（5 次/10 分钟）触发 IP 限流（接入服务器 fail2ban 或应用层计数）
- **F0.6** 所有写接口（增/删/改/排序/分类管理/访客管理）必须为 Owner，访客调用返回 403
- **F0.7** 退出登录清除 Cookie，并使后端 token 失效（Redis 黑名单 / 短有效期方案）
- **F0.8** 一期不做注册功能（无开放注册入口）

#### F1. 顶部 Tab 导航
- **F1.1** 顶部展示所有分类 Tab，初始数据为「互联网项目管理」「金融投资」「思维认知」
- **F1.2** 当前选中态高亮（颜色 / 下划线）
- **F1.3** Tab 切换无刷新，平滑过渡
- **F1.4** 默认进入第一个 Tab；URL 支持锚点（如 `/#/finance`），刷新保持当前 Tab
- **F1.5** Tab 数量与名称**动态可配置**（见 F9 分类管理）
- **F1.6** Tab 数量较多时（>6 个）支持横向滚动或溢出菜单

#### F2. 经验卡片列表
- **F2.1** 每个 Tab 内以**卡片网格**或**列表**形式展示该领域下的所有经验文档
- **F2.2** 每张卡片包含字段：
  - 标题（必填）
  - 简介 / 摘要（选填，最多 100 字）
  - 封面图或图标（选填，缺省用主题图标）
  - 上传时间 / 最近更新时间
  - 操作区：打开 / 编辑 / 删除 / 拖拽手柄
- **F2.3** 点击卡片主体 → 在新标签页或弹窗中打开对应 HTML 文档
- **F2.4** 列表为空时展示空状态引导（"还没有内容，点击右上角新增"）

#### F3. 新增经验
- **F3.1** 每个 Tab 右上角提供「+ 新增」按钮
- **F3.2** 点击后弹出表单：
  | 字段 | 类型 | 必填 | 说明 |
  | --- | --- | --- | --- |
  | 标题 | 文本 | 是 | ≤ 50 字符 |
  | 简介 | 文本 | 否 | ≤ 100 字符 |
  | 封面图 | 图片上传 | 否 | 支持 jpg/png/webp，≤ 5MB |
  | HTML 文档 | 文件上传 | 是 | 仅 .html 文件，≤ **20MB**（原 5MB 已放宽） |
  | 所属分类 | 下拉 | 是 | 默认为当前 Tab |
- **F3.3** 上传成功后卡片出现在列表**末尾**
- **F3.4** 上传失败有明确错误提示（格式错误 / 超大 / 网络异常）

#### F4. 编辑经验
- **F4.1** 卡片操作区点击「编辑」→ 复用新增表单，预填当前值
- **F4.2** HTML 文档支持替换上传（不替换则保留原文件）
- **F4.3** 保存后更新"最近更新时间"

#### F5. 删除经验
- **F5.1** 点击「删除」弹出二次确认弹窗（"确认删除《xxx》？此操作不可恢复"）
- **F5.2** 确认后从列表移除，同时**服务器同步删除**对应 HTML 文件、封面图与元数据
- **F5.3** 删除操作支持**5 秒撤销**（toast 形式）【可选优化，依赖软删除字段】

#### F6. 拖拽排序
- **F6.1** 卡片支持鼠标按住拖拽（PC 端）/ 长按拖拽（移动端）
- **F6.2** 拖拽过程中有占位提示（虚框 / 阴影）
- **F6.3** 释放后立即更新列表顺序，并同步保存到服务器
- **F6.4** 排序仅在当前 Tab 内生效，跨 Tab 不允许拖动
- **F6.5** 排序失败时回滚到原顺序并提示

#### F7. HTML 文档查看
- **F7.1** 点击卡片在新标签页打开 HTML 文档
- **F7.2** HTML 通过后端签发的短期 Token URL 访问，避免直接公开
- **F7.3** 渲染时保持原文档样式与脚本（沙箱化处理，见安全章节）

#### F8. 服务端存储（部署于腾讯云轻量应用服务器）
- **F8.1** 所有 HTML 文件、封面图存于服务器**本地文件系统**（独立目录，按用户/分类组织）
- **F8.2** 元数据（标题/简介/排序/分类/时间）存于服务器**本地数据库**（PostgreSQL 或 SQLite）
- **F8.3** 支持后续多设备访问（同一服务，浏览器登录即可）
- **F8.4** 文件目录结构建议：
  ```
  /var/app/experience/
  ├── data/              # 数据库文件（SQLite 时）/ 或外挂 PostgreSQL
  ├── storage/
  │   ├── html/          # HTML 文档（按 experience_id 命名，如 {id}.html）
  │   └── covers/        # 封面图（按 experience_id 命名）
  └── backups/           # 自动备份（见 F11）
  ```
- **F8.5** 文件访问通过**后端服务签发短期 Token URL**（不直接 Nginx 静态映射），保障鉴权
- **F8.6** 服务器磁盘空间监控：剩余 < 20% 时后台日志告警

#### F9. 分类（Tab）管理（新增）
- **F9.1** 仅 Owner 可见入口：顶部导航右侧「分类管理」按钮
- **F9.2** 进入管理页可查看所有分类，支持：
  - **新增分类**：填写"名称"（≤ 20 字）、"图标"（可选 emoji 或图标库选择）、"slug"（URL 标识，自动生成可手改）
  - **编辑分类**：修改名称、图标、slug
  - **删除分类**：二次确认，**若分类下仍有经验文档则禁止删除**，提示"请先迁移或删除该分类下的内容"
  - **拖拽排序**：调整 Tab 在顶部的展示顺序
- **F9.3** 分类初始种子数据三条：互联网项目管理 / 金融投资 / 思维认知
- **F9.4** 分类的 slug 用于 URL 锚点（如 `/#/pm`），删除/修改 slug 需提示"将影响已分享的链接"
- **F9.5** 分类数量上限建议 ≤ 20（软限制 + 提示）

#### F10. 访客管理（新增）
- **F10.1** Owner 在用户菜单进入「访客管理」页面
- **F10.2** 列表展示所有访客账号：用户名 / 备注 / 状态 / 创建时间 / 最近登录时间
- **F10.3** 支持新增、禁用/启用、重置密码、删除
- **F10.4** 新增访客时自动生成 12 位随机密码，**仅在创建时一次性展示**
- **F10.5** 访客登录后只能看到列表 + 打开 HTML，不能看到任何写操作入口

#### F11. 数据备份（建议加入，新增）
- **F11.1** 后端定时任务（如每天凌晨 3 点）自动打包 `storage/` + 数据库导出到 `backups/`
- **F11.2** 保留最近 7 份，自动清理过期备份
- **F11.3** 后台提供"立即备份"与"下载最新备份"两个按钮（仅 Owner）
- **F11.4** 关键操作（删除经验、删除分类）后端落库前先记录操作日志，便于人工追溯

## 四、非功能需求

### 4.1 性能
- 首屏加载 ≤ 2s（弱网 4G）
- Tab 切换 ≤ 200ms
- 单次拖拽排序保存 ≤ 500ms
- 单文件上传：20MB HTML 文件 ≤ 30s（轻量服务器带宽建议 ≥ 5Mbps）
- **建议开启上传分片 + 断点续传**，避免大文件上传中断需要从头开始

### 4.2 兼容性
- 主流浏览器最新两个版本：Chrome、Edge、Safari
- 响应式布局：桌面端为主，平板可用，手机端可读不强制可编辑

### 4.3 可用性
- 所有破坏性操作（删除、覆盖上传）需二次确认
- 错误提示统一使用 toast，文案明确
- 关键操作（上传、删除、排序保存）有 loading 态

### 4.4 安全性（重要）

> 由于核心功能涉及**用户上传 HTML 并在浏览器中渲染**，且部署在自建服务器上，必须做好安全防护。

| 风险 | 防护措施 |
| --- | --- |
| **XSS（HTML 内含恶意脚本）** | HTML 文件**用独立子域 + iframe sandbox** 加载（如主站 `app.example.com`，HTML 走 `view.example.com`），sandbox 设为 `allow-scripts`，**禁用 `allow-same-origin`**，与主站隔离 Cookie/LocalStorage |
| **HTML 文件大小炸弹** | 上传时校验大小 ≤ 20MB；服务端二次校验；Nginx 层 `client_max_body_size 25m` |
| **文件类型伪造** | 上传时校验 MIME + 文件后缀 + magic number 三重校验，仅允许 `text/html` |
| **存储路径遍历** | 文件名服务端重写为 `{uuid}.html`，禁止使用用户上传的原始文件名做磁盘路径 |
| **静态文件越权下载** | 不开放 Nginx 直接映射 `/storage/`；所有文件下载经后端鉴权后签发**短期 Token URL**（如 5 分钟有效） |
| **登录爆破** | 登录失败 5 次/10 分钟锁定 + 服务器层 fail2ban 联动 |
| **会话劫持** | JWT 走 HttpOnly + Secure + SameSite=Strict Cookie；HTTPS 全站强制 |
| **CSRF** | 写接口校验 Origin/Referer + 双 Token（Cookie + Header） |
| **接口鉴权** | 所有写操作（增/删/改/排序/分类/访客）需校验角色 = Owner，访客返回 403 |
| **SSRF** | 后端不主动拉取用户提供的任意 URL；HTML 内若发起外部请求，由浏览器侧承担 |
| **敏感信息泄露** | 数据库密码、JWT Secret、初始密码哈希等仅放服务器 `.env`，不进 Git；`.env` 权限 `chmod 600` |
| **封面图上传** | 限制图片格式与尺寸；服务端校验后做格式标准化（去除 EXIF 元数据） |
| **服务器入侵** | 关闭 root SSH 登录、改默认 SSH 端口、UFW 仅放行 80/443/SSH；定期更新系统补丁 |
| **数据丢失** | F11 自动备份；建议每周手动下载一份本地冷备 |
| **HTTPS** | 使用 Let's Encrypt 免费证书 + Certbot 自动续期；HSTS 开启 |

### 4.5 可维护性
- 数据模型预留扩展字段（tags、is_pinned 等）
- Tab 配置结构化，未来可改为后台可配
- 前后端接口遵循 RESTful，版本化（如 `/api/v1/...`）

## 六、接口设计

### 6.1 认证
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/v1/auth/login` | 登录（用户名 + 密码） |
| POST | `/api/v1/auth/logout` | 退出登录 |
| GET | `/api/v1/auth/me` | 获取当前用户信息（含 role） |

### 6.2 分类管理
| 方法 | 路径 | 说明 | 权限 |
| --- | --- | --- | --- |
| GET | `/api/v1/categories` | 获取分类列表（按 order 升序） | 登录 |
| POST | `/api/v1/categories` | 新增分类 | Owner |
| PUT | `/api/v1/categories/:id` | 编辑分类 | Owner |
| DELETE | `/api/v1/categories/:id` | 删除分类（非空时拒绝） | Owner |
| PATCH | `/api/v1/categories/reorder` | 批量更新分类顺序 | Owner |

### 6.3 经验文档

### 7.2 关键交互
- **拖拽排序**：鼠标移到卡片上显示 ⠿ 手柄 → 按住拖动 → 其他卡片实时让位 → 释放后保存
- **新增 / 编辑**：右侧滑出抽屉或居中模态框，避免离开当前 Tab 上下文
- **删除**：二次确认 → toast 提示成功 + 撤销按钮（5s）
- **分类管理**：进入独立管理页（或抽屉），新增/编辑/拖拽 Tab；删除有内容的分类时直接拒绝并提示
- **访客视图**：登录态为 visitor 时，前端隐藏所有写操作 UI；后端再做一次校验

## 八、技术选型建议（参考，非强制）

| 层 | 建议方案 | 理由 |
| --- | --- | --- |
| 前端 | React + Vite + TypeScript | 生态成熟、构建快 |
| UI | Ant Design / shadcn-ui | 卡片、抽屉、表单组件齐全 |
| 拖拽 | dnd-kit | 比 react-beautiful-dnd 更现代、更轻量，支持卡片+Tab 双场景 |
| 状态管理 | Zustand / TanStack Query | 简单够用 |
| 后端 | Node.js (NestJS) **或** Python (FastAPI) | 任选熟悉的，单进程即可 |
| 数据库 | **SQLite**（一期推荐，零运维）/ PostgreSQL（数据量大后再切） | 关系型，便于排序与扩展 |
| 文件存储 | **服务器本地文件系统**（轻量服务器 SSD） | 部署简单，无外部依赖 |
| 认证 | JWT + bcrypt + HttpOnly Cookie | 标准方案 |
| Web Server | Nginx（反向代理 + HTTPS + 静态前端） | 轻量服务器标配 |
| 进程管理 | PM2（Node）/ systemd（通用） | 守护后端进程，开机自启 |
| HTTPS | Let's Encrypt + Certbot 自动续期 | 免费 + 自动 |
| HTML 沙箱 | 独立子域 `view.example.com` + iframe sandbox | 隔离主站，防 XSS |
| 部署 | 全栈部署在腾讯云轻量应用服务器（前端构建产物 + 后端 + 数据库 + 文件） | 单机方案，符合需求 |

### 8.1 部署架构图（单机方案）

```
                  ┌───────────────────────────────────────┐
                  │  腾讯云轻量应用服务器 (Ubuntu 22.04)    │
                  │                                       │
   用户浏览器 ───► │  Nginx :443 (HTTPS)                   │
                  │  ├── app.example.com  → 前端静态文件   │
                  │  ├── api.example.com  → 反代到 :3000   │
                  │  └── view.example.com → 反代到 :3000   │
                  │                          (HTML 渲染)   │
                  │                                       │
                  │  Node/Python 后端 :3000               │
                  │  └─► SQLite (data/app.db)             │
                  │  └─► storage/ (html/ + covers/)       │
                  │                                       │
                  │  Cron: 每日 03:00 备份 → backups/     │
                  └───────────────────────────────────────┘
```

> **关键要点**：主域 `app.example.com` 与 HTML 渲染域 `view.example.com` 必须是**不同二级域名**，浏览器基于同源策略隔离 Cookie/LocalStorage，是 XSS 防护的核心。两个域名都解析到同一台轻量服务器即可。

### 仍待确认 / 建议补充

1. **域名**是否已就绪？需要至少 2 个二级域名（主站 + HTML 渲染域），如 `app.xxx.com` + `view.xxx.com`。如未备案，是否先用 IP + 端口临时跑通？
2. **后端语言偏好**：Node.js（NestJS）还是 Python（FastAPI）？
3. **前端 UI 库偏好**：Ant Design（中文友好、更稳重）还是 shadcn-ui（更现代）？
4. **登录页是否需要找回密码**？一期可不做，靠站主直接 SSH 改库。
5. **是否需要标签（Tag）作为二级筛选**？一期建议不加，但可在数据模型预留 `tags` 字段。
6. **轻量服务器配置**：CPU/内存/磁盘多少？SQLite + 单进程后端在 2C2G 即可流畅运行；如经验文档积累很多（>1000 篇），建议 4G 内存 + 50G SSD 起。
7. **备份是否需要异地存储**？建议至少周度同步一份到本地或 COS，防止服务器单点故障。

## 附录 A：术语表
- **Tab**：顶部一级分类入口
- **经验卡片**：单篇 HTML 文档在列表中的展示单元
- **沙箱（Sandbox）**：使用 iframe sandbox 属性 + 独立子域隔离不受信任 HTML 的运行环境
- **Owner / Visitor**：站主（全权限）/ 访客（只读）
- **签名 Token URL**：后端鉴权后签发的短期文件访问链接，避免静态文件被公开访问
